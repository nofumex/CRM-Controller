# Мониторинг клиентских сделок amoCRM в Telegram

Бот показывает, где сейчас находятся клиентские сделки, пришедшие из разных источников. Telegram никогда не запрашивает amoCRM в момент нажатия кнопки: интерфейс читает последний готовый снимок из SQLite, поэтому разделы открываются сразу.

## Источники клиентов

- **Клиенты по судебному приказу** — сделки вне воронки `Судебный приказ`, у контактов которых совпадает телефон с телефоном контакта сделки из этой воронки.
- **Заявки от агентов реферальной программы** — сделки, которые когда-либо находились в воронке `TG / Max - Боты`, но уже вышли из неё. Пока сделка находится на любом этапе исходной воронки, бот её не показывает.
- **Контакты от юристов** — сделки, которые когда-либо находились в воронке `[А7] TG / Max - Юристы`, но уже вышли из неё.

В каждом источнике сделки разделены по их текущим воронкам и этапам. Этапы отдела продаж и прогрева объединяются в удобные разделы так же, как в прежнем экране TG/MAX.

Внутри этапа сделки показаны отдельными кнопками по 5 на страницу. Карточка сделки содержит текущий и начальный этапы, ответственного, бюджет, даты и хронологию переходов; отдельная кнопка открывает сделку в amoCRM. Эту же карточку можно получить, отправив боту ссылку вида `https://example.amocrm.ru/leads/detail/31094071` или просто ID `31094071`. Сделки мониторинга открываются из локального кэша, а информация о любой другой сделке загружается по запросу и затем также кэшируется.

## Обновление данных

- Первый снимок начинает формироваться в фоне сразу после запуска бота.
- Новый цикл начинается каждые 15 минут.
- Нажатие кнопок не запускает синхронизацию с amoCRM.
- Снимки сохраняются в `bot_data.sqlite3` и доступны сразу после перезапуска.
- Если очередное обновление завершилось ошибкой, бот продолжает показывать предыдущий успешный снимок и отмечает это на экране.

Бот больше не перемещает сделки, не назначает менеджеров и не отправляет уведомления о таких действиях. Соответствующих кнопок и настроек в интерфейсе нет.

## Настройка

Создай `.env` на основе `.env.example`:

```env
AMOCRM_BASE_URL=https://example.amocrm.ru
AMOCRM_ACCESS_TOKEN=your_access_token
TELEGRAM_BOT_TOKEN=123456:telegram_token
TELEGRAM_ADMIN_IDS=123456789
```

Несколько Telegram ID можно указать через запятую.

## Запуск

```powershell
python main.py
```

Часовой пояс отображаемого времени: `Asia/Krasnoyarsk` (GMT+7).
