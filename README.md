# amoCRM Telegram bot

Бот каждый день по красноярскому времени переводит сделки из выбранной воронки и этапа в выбранную воронку и этап, распределяя их по менеджерам.

## Что умеет

- Автоперевод по расписанию, по умолчанию в `09:00` Красноярск.
- Ручной запуск из Telegram.
- Админ-панель в Telegram:
  - менять менеджеров;
  - менять количество сделок на каждого;
  - менять исходную и целевую воронку/этап;
  - менять время запуска.
- Мониторинг:
  - сколько всего переведено ботом;
  - последние запуски;
  - распределение по менеджерам;
  - сколько сейчас сделок в исходном и целевом этапах.
- Раздел сделок из TG/Max:
  - показывает этапы, где сейчас находятся сделки, которые когда-либо уходили из исходной воронки по событиям amoCRM;
  - внутри каждого этапа показывает сделки со ссылками на amoCRM.
  - технические/закрывающие этапы можно показывать объединённой кнопкой, например `Отдел продаж` или `Прогрев базы`.
- Защита от повторного запуска:
  - бот не даст выполнить вторую итерацию переводов в течение 24 часов;
  - максимум за итерацию равен сумме лимитов активных менеджеров.

## Настройка

1. Создай бота через `@BotFather` и получи `TELEGRAM_BOT_TOKEN`.
2. Узнай свой Telegram ID. Можно написать боту без `TELEGRAM_ADMIN_IDS`, он ответит твоим ID.
3. Добавь в `.env`:

```env
TELEGRAM_BOT_TOKEN=123456:telegram_token
TELEGRAM_ADMIN_IDS=123456789
```

`AMOCRM_BASE_URL` и `AMOCRM_ACCESS_TOKEN` уже используются из текущего `.env`.

## Запуск

```powershell
python main.py
```

Бот хранит настройки и историю в `bot_data.sqlite3`. Файл создаётся автоматически при первом запуске.
Если локальная база была удалена во время работы, бот восстановит таблицы при следующем обращении к настройкам.

## Настройки по умолчанию

- Источник: `TG / Max - Боты` → `В работе у менеджера`
- Назначение: `Отдел продаж` → `НЗ-2`
- Менеджеры:
  - Ольга Шевелева: 15
  - Дегтярева Юлия: 15
  - Павел: 15

Сделки выбираются с конца по номеру в названии: сначала самый большой `#`, затем ниже.

## Важное про историю

Раздел `Сделки из TG/Max` синхронизируется с событиями amoCRM: бот ищет сделки, у которых до смены этапа была исходная воронка, и показывает, где эти сделки находятся сейчас. Для скорости история кэшируется в `bot_data.sqlite3` и обновляется кнопкой `Обновить`.
